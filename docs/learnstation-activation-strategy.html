01 / Strategic frame

Two jobs. Two finish lines.

Do not teach every feature. Ask one intent question, remove irrelevant setup, and lead each persona to one observable outcome.

ASCEND is not another content library. It is a confidence engine that converts trusted course material into visible learning progress for students and actionable teaching evidence for professors.

Student

Learn efficiently & compete

Entry intent: pass an exam, understand a topic, or join a course.

  • Promise Answers grounded in their lecture material
  • Primary path Select course → Skill Tree → Tutor → quiz
  • Risk Empty-state confusion; skepticism about AI accuracy
  • Hook Visible mastery, XP, friends, weekly rank
Professor

Track engagement & diagnose content

Entry intent: publish materials, understand confusion, improve teaching.

  • Promise See where students struggle, slide by slide
  • Primary path Create course → upload → preview tree → invite
  • Risk Parsing delay; no data before students arrive
  • Hook Actionable alerts, adoption coverage, cohort trends
The strategic wedge

Trust first. Structure second. Social third.

ASCEND wins when its differentiators appear as one connected value chain rather than five competing features.

1 / Ground

Trusted answer

The Socratic Tutor answers only from course sources and cites the exact page. This removes the first adoption objection: “Can I trust it?”

2 / Orient

Visible path

The Skill Tree turns disconnected lectures into a navigable mastery map. It answers: “What should I learn next?”

3 / Prove

Measured progress

Short checks convert explanation into evidence. Mastery, not consumption, triggers activation and XP.

4 / Compound

Shared signal

Student behavior becomes professor insight; course improvement creates better future learning and stronger retention.

Segmentation logic

Role chooses the lane. Intent chooses the next step.

Do not create a separate onboarding for every cohort. Use role, readiness, and immediate intent to choose the next-best action dynamically.

Axis 01 / Role

Who receives value?

Student: learning confidence.
Professor: teaching visibility.
This determines the activation definition.

Axis 02 / Readiness

What exists already?

Student: course found, invite link, or no course.
Professor: material ready, course live, or evaluating.

Axis 03 / Urgency

Why today?

Exam prep, concept repair, exploration, course launch, engagement diagnosis, or content improvement.

Routing rules

Invited student → open shared course and recommended node. Searching student → find course by code; if absent, use a two-minute demo. Professor with material → create shell and upload immediately. Professor evaluating → preview populated analytics with clearly labeled sample data. Ask only the question needed to choose this branch.

Activation architecture

A ladder, not a checklist.

Each stage earns the right to ask for more effort. Social and retention mechanics begin only after core value is proven.

1. OrientRole + intent chosen; user sees a relevant path
2. TrustSource boundary and citation are understood
3. ProveStudent passes check / professor sees live insight
4. CompoundNext action, return cue, and contextual invite
Activation equation: relevant context + trusted output + completed outcome + clear next step. Remove any onboarding task that does not strengthen one of these four terms.
Strategic choices

What ASCEND will and will not optimize.

We will
  • Optimize for demonstrated learning, not content views
  • Show source evidence before celebrating AI novelty
  • Use one recommended node before exposing the full tree
  • Reward quality actions and activated invitations
  • Let background work remain resumable and transparent
We will not
  • Force a generic product tour before core value
  • Gate first value behind profile completion
  • Use raw XP to disguise weak learning outcomes
  • Show public ranking before users establish confidence
  • Count upload completion or account creation as activation
System-level flywheel

Every activated user improves the course.

The durable moat is the interaction between grounded learning activity and professor action, not any feature in isolation.

1. Publish trusted materialProfessor uploads and verifies sources.
2. Generate learning signalsStudents ask, practice, and reveal confusion.
3. Diagnose frictionAnalytics connect questions to slides and nodes.
4. Improve the courseProfessor updates material; learners return to a stronger path.

04 / Async friction

Turn processing time into progress.

Arq/Redis processing should feel resumable and purposeful. Never trap the user on a spinner or imply false precision.

The productive wait pattern

Upload in the background. Advance in the foreground.

1. Instant receiptFilename, pages, size, passed validation. “Safe to leave this screen.”
2. Stage progressUploading → extracting → mapping concepts → quality check. Use ranges, not fake %.
3. Parallel tasksOffer 2-minute jobs: set goal, name course, invite co-teacher, tour demo tree.
4. Re-entryIn-app + email notification deep-links to “Review Skill Tree.” Preserve every choice.
Under 30 secondsInline stage animation + “While we map 42 pages...” one-click goal chips.
30 seconds–3 minutesRelease user into setup checklist; persistent processing pill in header.
Over 3 minutes / failedOffer leave-and-notify. On failure: retain file, explain fix, retry from failed stage.
Guardrail: parsing completion is not activation. Track upload-to-review conversion and recover unfinished reviews within 24 hours.

3D Skill Tree onboarding

Use a “guided camera,” not a modal tour: spotlight one recommended node, label current/next/locked states, and let the first click teach pan/zoom. Always provide list view for accessibility and low-power devices.

Trust treatment for the AI Tutor

Lead with “Answers only from your course materials.” Show page/slide citations expanded on the first answer, a “not found in sources” refusal demo, and a report-answer control. Trust precedes query volume.

06 / Growth loops

Reward progress, not busywork.

XP marks meaningful learning. Social prompts appear only after earned value, and every invite carries shared context.

Checkpoint 01

First query

+25 XP for citing a source-backed answer. Achievement: “Curious Mind.”

Checkpoint 02

First passed quiz

+75 XP and “First Ascent.” Show rank only after 3 learning actions.

Checkpoint 03

Course publish

Professor earns “Cartographer.” Unlock a share card with course code and tree preview.

Checkpoint 04

Study ally joins

Both get +50 XP after the invitee completes one node; blocks spam invites.

Student retention loop

Master node → gain XP → see friend progress → accept next challenge → return for streak. Use weekly leagues of 10–20 comparable learners, opt-in by default during onboarding.

Professor growth loop

Publish course → invite cohort → collect questions → reveal confusion hotspots → improve material → publish update. Reward student coverage and resolved hotspots, not raw logins.
